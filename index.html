注意事项

基本配置：软件部署机器需要能够ssh免密码登陆发布目标机器，这是本项目可以运作的根本原因。

本地仓库路径：需以repos/开头，因为从仓库检出的代码存在在repos目录下。

发布用户：必须是可面密码登陆代码发布目标机器的用户，并且用户主目录下存在id_rsa文件。请使用命令ls $HOME/.ssh/检查

机器列表：支持多机发布，格式需严格遵守。以,分隔每台机器。以:分隔ip和端口。端口号即使是22也必须填写。

发布后任务：每行一条命令，会在发布结束后逐行执行。

测试链接按钮：当发布用户和机器列表填写后就可以进行ssh连通测试，确保填写了正确的发布用户和目标机器地址。

git版本：需要执行git clone -q git仓库地址 目标目录命令，所以请使用大于v2.0.0以上的git客户端版本，目前测试centos7为1.7.*版本，所以请重新安装git客户端。附上git客户端安装教程

最后，请保证具有git仓库的检出权限

最后一步

将数据库user表内自己的role设置为2

update user set role=2

配置数据库和项目监听端口,在项目根目录找到配置文件config.json

{ "Listen": ":8088", "DBIp": "127.0.0.1", "DBPort": "3306", "DBUser": "root", "DBPass": "password", "DBName": "gopub", "SqlLog": true }
